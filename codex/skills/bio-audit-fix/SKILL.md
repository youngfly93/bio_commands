# 审计-修复一体化循环

不要只做计划，立即执行修复。

## 审计维度

对每个维度逐一检查（与 $bio-check 一致）：

1. 完整性：plan.md 中每个分析步骤是否都有对应结果文件
2. 数据准确性：抽样验证关键数值
3. 分析准确性：统计方法、参数、阈值
4. 方法合理性：工具/流程/参考基因组
5. 图表-数据一致性
6. 基因组版本合规

## 问题分级

- **P0 — 数据完整性**：缺失结果、数据丢失、关键文件损坏
- **P1 — 方法正确性**：参数错误、工具版本不对、基因组版本不匹配
- **P2 — 完整性缺失**：部分分析未完成但不影响核心结论
- **P3 — 润色优化**：图表美化、报告格式、命名规范

## 执行流程

### 每一轮循环：

1. **审计**：按 $bio-result-auditor 的逻辑全面审计，输出问题清单
2. **分级**：按 P0→P3 排序
3. **修复**：立即修复，P0 优先
   - 缺失分析：重新运行脚本
   - 数据不一致：追溯源头修正
   - 图表问题：重新生成
   - 代码问题：直接修改脚本
4. **提交**：`git add -A && git commit -m "audit-fix round N: 修复内容摘要"`
5. **重新审计**：验证修复是否生效，检查是否引入新问题

### 循环控制

- 最多 **5 轮**
- 当所有 P0/P1 问题清零且无新问题时提前终止
- 每轮开始前输出当前轮次和剩余问题数

## 最终输出

审计进展矩阵：

```
| 轮次 | P0 | P1 | P2 | P3 | 状态 |
|------|----|----|----|----|------|
| R1   | 3  | 5  | 2  | 4  | 修复中 |
| R2   | 0  | 2  | 2  | 4  | 修复中 |
| R3   | 0  | 0  | 1  | 3  | 通过 ✅ |
```

剩余未修复问题（如有）及原因说明。
