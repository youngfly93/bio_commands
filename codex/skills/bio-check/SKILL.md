# 生信结果一次性审计

调用 $bio-result-auditor 的审计逻辑，对照 `plan.md` 检查以下维度，输出结构化审计报告：

1. **完整性**：plan.md 中要求的每个分析步骤是否都有对应结果文件
2. **数据准确性**：结果文件中的数据是否与原始数据/中间文件一致（抽样验证关键数值）
3. **分析准确性**：统计方法、参数设置、阈值选择是否正确
4. **方法合理性**：使用的工具/流程/参考基因组版本是否合适
5. **图表-数据一致性**：图表展示的数据是否与结果文件吻合
6. **基因组版本**：参考基因组/注释版本是否与 plan.md 要求一致

## 输入

- 当前项目根目录（默认 `.`）
- `plan.md`（必需）
- `results/`、`figures/`、`scripts/`、`reports/` 等标准目录

## 输出

按 $bio-result-auditor 中定义的审计报告模板输出，包含：

- 总体评价
- Plan.md 需求覆盖情况表（✅/⚠️/❌）
- 结果溯源与分析逻辑评估
- 主要不足与风险
- 改进建议

## 风格

实事求是，发现问题直接指出，明确区分"事实/推断/建议"。

## 仅审计、不修改

本 skill 只读，不会运行分析、不会修改任何文件。如需修复，使用 $bio-audit-fix。
