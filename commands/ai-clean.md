扫描并清除项目交付文件中的 AI 使用痕迹。

## 扫描范围

### 1. Office 文档元数据（.docx/.xlsx/.pptx）

解压 Office 文件检查 XML 元数据：

```bash
# 解压到临时目录检查
unzip -o "$file" -d /tmp/office_check/
```

检查以下文件中的字段：
- `docProps/core.xml`: `dc:creator`, `cp:lastModifiedBy`
- `docProps/app.xml`: `Application`, `AppVersion`

需要清除的关键词：
- creator/lastModifiedBy 中包含: `Claude`, `Anthropic`, `GPT`, `OpenAI`, `ChatGPT`, `AI Assistant`, `Copilot`, `Gemini`, `Google AI`
- Application 中包含: `Claude`, `AI`

### 2. 文本内容 AI 典型表达

扫描 .docx 的 `word/document.xml` 和其他文本文件：

**中文关键词**:
- `值得注意的是`, `需要指出的是`, `综上所述`, `总而言之`
- `作为一个AI`, `作为人工智能`, `语言模型`
- `我无法`, `我没有能力`, `我的训练数据`
- `根据我的分析`, `基于上述分析`

**英文关键词**:
- `As an AI`, `language model`, `I cannot`, `I don't have the ability`
- `It's worth noting`, `It should be noted`, `In conclusion`
- `Based on my training`, `As of my knowledge cutoff`

### 3. 代码文件注释

扫描 .R/.py/.sh 文件中的注释：
- `# Generated by`, `# Created by AI`, `# Claude`, `# GPT`
- `# This code was generated`, `# Auto-generated`

## 输出发现汇总

```
| # | 文件 | 类型 | 发现内容 | 位置 |
|---|------|------|----------|------|
| 1 | report.docx | 元数据 | creator: Claude | core.xml |
| 2 | report.docx | 内容 | "值得注意的是" | 第3段 |
| 3 | analysis.R | 注释 | "# Generated by Claude" | 第1行 |
```

## 修复流程

1. **备份**: 修改任何 .docx 前先复制为 `.docx.bak`
2. **元数据修复**: 修改 XML 中的字段值为合理替代（如用户真实姓名）
3. **内容修复**: 将 AI 套话替换为更自然的学术表达
4. **代码注释修复**: 移除或替换 AI 相关注释
5. **重新扫描**: 验证所有问题已修复

## 注意事项

- 仅修改明确的 AI 痕迹，不要改变文档的实质内容
- 元数据修改后需要重新打包 ZIP 为 .docx
- 修复内容表达时保持学术文风的一致性
